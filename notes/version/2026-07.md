## v0.6.0 — 2026-07-01

Daily maintainer run. **Maintain:** no open issues/PRs; baseline 110/110 green;
cleared a stale `next-steps.md` (its Ricochet-registration items were long done).

- **New game — Skyline** (`games/skyline/`): a stack-and-slice tower builder,
  mechanically distinct from every existing game. A slab slides above your tower; drop
  it (one control) and only the overlap with the slab below stays — the overhang is
  sliced off, so every sloppy drop narrows your target. A **flush** drop keeps the full
  width and pays double, so precision is the only way to keep climbing; a total miss
  topples the run; the slide speeds up with height. Whole simulation is a pure core
  (`drop`, `tick`, `moveCurrent`, `spawnCurrent`, `milestoneBetween`) — notably **no
  timer-driven death** (a slab only resolves on an explicit drop). Full quality bar:
  core + 18 tests + shell + index (boot-failure fallback, guarded loop) + README +
  package.json. Browser-verified (renders, drops, slices, no console errors).
- **Grew Polarity** — **clutch saves**: a match landed by a last-moment flip (within
  `CLOSE_TICKS`) is tallied by the pure core (`isClutch`, `g.clutch`) and surfaced on
  the game-over card — a measure of how much you played on the edge, with no play-field
  clutter. +6 tests (20 → 25). How-to/tip refreshed.
- Registered Skyline in the landing `index.html` grid (tags **Timing**, **Precision**)
  and footer, bumped the masthead **Games** count to **6**, and added the README "The
  games" row. Also backfilled the footer's missing **Ricochet** link.
- **133/133 tests** green (was 110). New/modified shells `node --check`-clean. Released
  **dev → main** as a tagged `--no-ff` **MINOR** via `release/0.6.0`.
