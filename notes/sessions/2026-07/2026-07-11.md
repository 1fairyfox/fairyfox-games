### Design — the verb

Verbs already used: steer, time-a-catch, thrust/physics, flip-match, aim-bounce, precision-stack,
keep-aloft/rhythm, balance, mirror-coordination, judge-power/charge, sort/route. **Tether takes
swing/grapple** — a real pendulum, and the collection's first. Anchors hang ahead across an endless
sky; hold to rope onto one and swing, release to fly, miss and you fall past the floor.

**The hook fell out of the physics rather than being bolted onto it.** Exit velocity is the swing's
tangential velocity, so *the release angle is the launch angle* — letting go becomes a pure
projectile trade-off (fast-and-flat vs slow-and-high), with the classic ~45° launch as the sweet
spot. The **whip** both grows the multiplier and *boosts the launch*, so it isn't a scoreboard
bolted on the side: it is literally the distance that clears the next gap. Skill and survival are
the same act. Holding **pumps** the swing higher — hold to wind up, let go to launch.

### The physics fought back (three times)

Simulated bots (a scratch harness, never committed) caught all of it before a line of shell code
existed — the pure-core split earning its keep:

1. **First design was unwinnable.** The original hook was "release just past the bottom of the arc"
   — but that launches you *flat*, so you only ever descend, and the next anchor is always above
   your reach. Every bot died at 3 anchors. Working the algebra out, flight range is
   `2·L·(1−cos amplitude)` — **gravity cancels** — so distance is decided by *where in the arc* you
   let go. Moved the window up the forward swing to straddle 45°.
2. **Runs bled energy and strangled themselves.** A physically-honest rope eats the radial part of
   your speed on every catch; amplitude decayed until the swing could no longer reach the whip
   window and the player hung there, unable to whip and unable to die. Made the catch
   **speed-conserving** (the rope whips you round rather than absorbing you).
3. **The pendulum looped over the top, then froze solid.** A flat ω clamp (`OM_MAX`) sat *above*
   the over-the-top threshold `2√(g/L)`, so a fast catch on a long rope sent θ running away
   unbounded (θ ≈ −28 rad observed) — a run that could neither progress nor end. Replaced with an
   **energy cap** (`maxOmega`, amplitude ≤ `AMP_MAX` < π/2). That in turn pinned ω at 0 *outside*
   the cap angle and froze the pendulum mid-air — so the cap now only binds inside the cap angle,
   and `MIN_DROP`/`GRAB_R` are set so no legal catch can land outside it.

All three are pinned by regression tests. Both failure modes looked identical from outside — alive
but going nowhere — so the soak test asserts the honest invariant: **if a run is still breathing,
it's because it's thriving, not because it's stuck** (a flawless bot legitimately flying forever is
a skill ceiling, not a bug, so "every run must die" would have been the wrong assertion).

### Deepened: Ricochet

Ricochet had the least varied-structure coverage of the remaining four (Ricochet, Skyline,
Loft, Poise) and the most natural fit: its field was a flat random sprinkle
(`spawnTarget` rejection-sampled a point in the upper box), so every run offered the same
textureless spread of angles — exactly the "played once = played always" complaint.

The field is now a seeded **sequence of named target layouts** from a stage-weighted pool
(`FORMATIONS` / `pickFormation` / `loadFormation`, copied in shape from Polarity into
Ricochet's own core — self-contained, no shared runtime):

| Layout | minStage | Character |
|---|---|---|
| **Scatter** | 0 | the calm on-ramp — the classic loose spread |
| **Rack** | 0 | a billiards break: a tight triangle — thread it for a huge bank, or waste the shot |
| **Gallery** | 1 (notable) | a row of targets at one height — one flat shot can sweep the lot |
| **Ladder** | 1 (notable) | a diagonal climb; each pickup drags the next shot further out |
| **Pockets** | 2 (notable) | tucked high against the side walls — only a bank reaches them |
| **The Gauntlet** | 2 (notable) | the dense late crescendo: packed along both walls + the ceiling |

- **Progression drives the variety:** `minStage` gates each layout, so climbing the stages
  *opens the pool*; `weight(stage)` leans the late run onto the bank-only ones (the calm
  share falls from >75% at Rookie to <40% at Bank master — pinned by a test).
- **Slots are fractions, not pixels.** A layout returns `{fx, fy}` specs; a new pure
  `placeSpec` resolves one to the field — clamps into the legal spawn box, lifts it clear
  of the launcher (`SPAWN_CLEAR`), and nudges it off any target it would overlap. So a
  layout reads cleanly at *any* target radius and the per-stage shrink still layers on top.
  The old rejection-sampling loop is gone.
- **Legibility:** `fire()` returns a `formation` cue; the shell flashes `#formCue` for the
  notable layouts only (the calm ones pass silently), honouring `prefers-reduced-motion`.
  Start-panel copy names the set so a new player knows runs vary.
