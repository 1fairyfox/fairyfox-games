## v0.26.0 — PLANT: Loom — a new verb, **weave / interlace**

**Why a new game.** PLANT run (`fairyfox-games-new`). The collection had 16 distinct verbs;
the brief was a genuinely new one. Checked the used list (steer, time-a-catch, thrust,
flip-match, aim-bounce, precision-stack, keep-aloft, balance, mirror-coordination, charge,
pour, swing, recall, guard, herd, **sort/route**) — note Sluice already owns sort/route, so
the "route the spark" ideas were out. Landed on **weave**: the pegs have no right side of their
own; the weave is defined *relative to your own previous pass*. No other game is about building
an interlaced pattern. 17th game, first **weaving** mechanic.

**The mechanic (≈3s).** A shuttle lays the weft toward the loom line; warp pegs stream in. One
input sends the thread **over** or **under** each peg (click/Space toggles; ↑/↓ set the side
directly). Alternating side-to-side is a proper **interlace**; repeating a side is a **float**
(a flaw — the multiplier snaps to ×1). Gold **beads** on your side pay a bonus; red **barbs**
on your side are a **snag** (a life lost; three snags end the run).

**Depth inside the one verb, from birth** (per `../reference/depth-inside-the-mechanic.md`):

- **No plateau** — `speedOf` is a smooth asymptote on pegs woven (never goes dead-flat).
- **Cinch (hidden tech).** Flip to the interlacing side within `CINCH_TICKS` (5) of the peg and
  the weave cinches: `mult`++ plus `CINCH_BONUS`. Alternating *early* is safe but neutral — so
  the deep line is to commit every toggle late, which is scary (a mis-time floats or snags).
  Taught nowhere. (Polarity's precise/safe tension, re-expressed as over/under.)
- **Sheen (the reversal).** 3 cinches in a row → `SHEEN_TICKS` (~5s) where every point doubles —
  the most precise hand becomes the greediest.
- **Gossamer (secret stage).** Past Brocade (220 woven), named on no start screen.

**Varied structure + progression.** A run is a seeded sequence of named **drafts**
(`FORMATIONS`, `minStage`-gated, `weight(stage)`-biased so late runs lean hard): **Plain** (clean
on-ramp, no barbs) · **Basket** (beads clustered on one side — the greed window) · **Chevron**
(diagonal beads, barb on the empty side) · **Herringbone** (barbs on the interlacing side —
the real reads) · **Sateen** (sparse, tense) · **Snarl** (the crescendo). No stage-0 draft places
a barb (pinned by a test), so the on-ramp is safe to learn on. Stages: Warp → Weft → Twill →
Damask → Brocade → (secret) Gossamer.

**Meta.** `loom.meta` — plays / best / bestStage / bestMult / totals {pegs, points, cinches,
beads} + 15 skill-safe badges; legacy `loom.best` preserved. Run-report card on game over.
Coins **Silk** fun mode (1 coin, additive shimmer over the cloth, score untouched).

**Build.** Pure `loom.core.js` (no DOM/timers, injected RNG) + `loom.core.test.js` (**36**
assertions, all green) + external-module `loom.shell.js` (canvas cloth render, boot fallback,
guarded loop) + index/README/package + a code-rendered icon (canvas → headless Chrome, the
woven-grid app tile). Collection **773/773** green. Wired via `_games/loom.md`,
`_data/changelog.json` (player-facing "new"), `npm run gen:readme` (17 games). VERSION → 0.26.0.

## v0.25.4 — Arc: the PIN, ONSLAUGHT, the pad asymptote, and Pinhole

**Why Arc.** Two games still lacked the depth layer — Arc and Sluice. Arc (born 2026-07-09)
is the older of the two, and it had the exact plateau the sweep flags: the pad half-width
was a per-stage constant and the stages stop at Dead-eye (land 42) — past that the pad never
got finer again, so the whole ceiling was visible in ~2 min. Ricochet was the template (also
a turn-based, discrete-shot game, so its Dead-Centre/Blaze shapes port cleanly to the lob).

**The four items, all on the one charge-and-release verb, all safe to not know:**

- **The PIN (hidden tech).** `pinBandFor(cfg, hw)` = `max(PIN_ABS 3, hw × PIN_FRAC 0.16)` — a
  razor sub-window inside the drawn bullseye band (`BULLSEYE_FRAC` 0.34). A land within it of
  dead centre is a pin: +`PIN_BONUS` 2 over the bullseye, gold burst, a streak. A fraction of the
  pad (not an absolute px) so it self-scales as the pad shrinks and stays clearly tighter than the
  bullseye on every stage (checked in tests). Off-centre bullseye / miss silently breaks it.
- **ONSLAUGHT (the reversal).** `PIN_TRIGGER` 3 pins in a row → `g.onslaught = ONSLAUGHT_LANDS 3`;
  the next 3 scoring lands pay ×`ONSLAUGHT_MULT` 2. The triggering land is scored *before* the
  streak check, so it's never itself doubled; a miss breaks the pin streak but does **not** consume
  a lit window (the lost life already punishes it). Shell: pad centre band + projectile halo burn
  gold (colour-only; reduced-motion honoured), a "Pin! +2" / "Onslaught ×2!" toast.
- **No plateau.** `padHalfWidth(cfg, landed)` = stage hw × `(1 − HW_SHRINK_SPAN 0.28 · landed/(landed
  + HW_SHRINK_K 70))`, hard-floored at `HW_HARD_MIN` 20px. Used in `spawnTarget` for both the pad
  width and the placement window (so pads still fit). ×1 at land 0 (opening feel unchanged); keeps
  shrinking within a stage, so the old Dead-eye flat-line is gone. `lob` reads the stored `g.target.hw`,
  so bullseye + pin both use the shrunk width consistently.
- **Secret stage Pinhole.** A 6th STAGES entry at land 66, `secret: true`, hw 26 (< Dead-eye 32, so
  the "pads shrink each stage" test still holds), announced once on reach (gold toast). The start
  tip's stage ladder + formation-name list were removed and replaced with a curiosity hook, so the
  secret isn't spoiled.

**Meta / badges.** 3 new skill-safe badges — Pinpoint (a pin), Onslaught (3 pins running), Pinhole
(reach stage 5). `totals.pins` added to the meta (lossless normalize upgrade); `pins`/`onslaughts`
