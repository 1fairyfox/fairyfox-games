## Project born — fairyfox-games v0.1.0

Spun the daily-games idea out of the fairyfox.io homepage into its own dedicated mesh
node. Decided the **collection** is the project (not one repo per game), so this is a
monorepo of self-contained game folders.

- **Ink Bloom** engineered to standard: split a pure logic core (`ink-bloom.core.js`,
  JSDoc'd, no DOM) from the rendering shell, and wrote a 20-test suite. Found + fixed
  the original prototype's frame-one death bug (the starting trail was seeded with the
  newest point at index 0, putting an old collidable point on the head); added a
  regression test.
- Scaffolded the repo: landing page, README (contributions welcome), CONTRIBUTING,
  MIT, issue/PR templates, CI (`node --test`), Pages deploy workflow, notes system,
  mesh-aware `CLAUDE.md`.
- Registered hub-side as an `integrated` project (pinned beside the site card) in both
  registries; added a node page.
- Released **v0.1.0** (dev → main, tagged).
